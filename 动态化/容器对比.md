# 优缺点
## RN
### 核心原理
JS + Native Bridge，UI由Native组件渲染
### 优点
跨端复用率高；动态更新；生态成熟；React语法易上手
### 缺点
Bridge通信有延迟；复杂动画性能差；首屏加载慢
### 适用场景
广告卡片、活动页、落地页、重展示弱交互页面、需要快速迭代的业务
## Flutter
### 核心原理
Dart + Skia引擎，UI完全由Flutter渲染
### 优点
性能接近Native；跨端一致性好；动画流畅
### 缺点
包体大；引擎嵌入复杂；动态更新受限（需热重载方案）
### 适用场景
高性能、复杂UI、多端一致性要求高的业务（如电商首页、复杂广告）
## KMP + CMP
### 核心原理
Kotlin多平台共享业务逻辑，UI用Compose多平台
### 优点
共享业务逻辑；原生UI渲染；性能好
### 缺点
UI跨端复用率低；生态不如RN/Flutter
### 适用场景
业务逻辑跨端复用（算法、数据层），UI需原生体验的业务
## WebView
### 核心原理
在Native中嵌入浏览器组件
### 优点
实现简单；动态更新方便；生态成熟
### 缺点
性能差（尤其动画、滚动）；原生交互弱
### 适用场景
内容页、资讯流、简单广告落地页
## 小程序
### 核心原理
定制化WebView容器 + 框架（微信、支付宝等）
### 优点
生态强；无需安装App；动态更新
### 缺点
依赖平台；性能受限；能力受限
### 适用场景
平台内运营活动、轻量业务、引流场景
# 业务选择
1.业务迭代速度  
是否需要频繁上线新样式？  
是否需要服务端下发模板动态渲染？  

2.性能要求  
是否有复杂动画、滚动、3D效果？  
是否需要接近 Native 的性能？  

3.跨端一致性  
是否需要 Android/iOS/Web 一致的 UI？  

4.包体大小  
容器是否会显著增加包体？  

5.生态与团队能力  
团队是否熟悉 React、Dart、Kotlin？  

6.平台依赖  
是否依赖微信、支付宝等平台？  

7.用户触达  
用户是否需要安装 App 才能使用？
