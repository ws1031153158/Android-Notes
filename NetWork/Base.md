# 云服务器
本质上就是在机房里，给你分配了一台 Linux 电脑。  
本地通过 SSH 进行加密的远程终端连接就像你坐在服务器前面操作一样。  
## systemd
没有 systemd -> 终端关闭 -> 程序停止  
有 systemd -> 程序变为系统服务 -> 开机自动启动/崩溃自动重启
# 公网IP
就像你家的门牌号，全世界任何人都可以通过这个地址找到你的服务器
# SSH
Secure Shell，安全的远程终端协议。  
本质是在本地和服务器之间，建立一条加密的通信隧道，让你可以在本地操作远程电脑。  
一般通过密钥登录：私钥放在本地，公钥放在服务器（登录时服务器用公钥加密一段数据发给本地，本地用私钥解密，解密成功 → 证明你是合法用户，不需要输密码）
# Nginx
Nginx 是一个反向代理，在后端之前（用户请求 → Nginx → 后端）  
## 作用
1.统一入口：所有请求先到 Nginx  
2.转发请求：Nginx 再转给后端的 8000 端口  
3.后续扩展：负载均衡、HTTPS、静态文件  
4.保护后端：后端不直接暴露在公网
# 安全组
服务器的防火墙。  
默认所有端口都是关闭的，未开放端口无法外部访问，开放端口可指定访问方式（22-SSH、80-HTTP、443-HTTPS）
# 域名
IP地址的别名，更好记，更专业
# DNS解析
域名解析为IP地址，告诉全球的 DNS 服务器这个域名对应这个 IP
